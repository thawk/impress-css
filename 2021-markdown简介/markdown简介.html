Markdown简介

* ``Markdown``是一种轻量级标记语言，创始人为``John Gruber``。它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的``XHTML``（或者``HTML``）文档。 * 由于``Markdown``的轻量化、易读易写特性，并且对于图片，图表、数学式都有支持，目前许多网站都广泛使用``Markdown``来撰写帮助文档或是用于论坛上发表消息。如``GitHub``、``Reddit``、``Diaspora``、``Stack Exchange``、``OpenStreetMap ``、``SourceForge``、``简书``等，甚至还能被用来撰写电子书。
* 最早版本是格鲁伯编写的``perl``脚本``Markdown.pl`` * 从2012年开始，包括``Jeff Atwood``和``John MacFarlane``在内的一群人启动了标准化工作。通过``CommonMark.org``发布了规范、参考实现和测试包的几个版本。 * 2016年3月发布了``RFC 7763``和``RFC 7764``。 - ``RFC 7763``从原始变体引入了MIME类型``text/markdown``。 - ``RFC 7764``讨论并注册了``MultiMarkdown``、``GitHub Flavored Markdown``(``GFM``)、``Pandoc``、``CommonMark``及``Markdown``等变体。 * 本幻灯片主要参考``GitHub Flavored Markdown ``(``GFM``) - [https://github.github.com/gfm/](https://github.github.com/gfm/)
### 空白字符 * **空白字符**包括 - space (``U+0020``)、tab (``U+0009``)、newline (``U+000A``)、line tabulation (``U+000B``)、form feed (``U+000C``)、carriage return (``U+000D``) * ``Tab``不会展开，Tab Stop视为4个空格 - 不同软件对``Tab``对处理不同，**避免使用Tab**

缩进代码块（idented code block）

  • 至少缩进4个空格。
  • 内容将去掉左边4个空格，保留右边空格
  • 代码块中的内容不会被解释
  • 代码块前后的空行不算入代码块中
正文

  正文，缩进2空格

   正文，缩进3空格

    代码，缩进4空格
    
     indented code
正文 正文，缩进2空格 正文，缩进3空格 代码，缩进4空格 indented code

包裹的代码块（fenced code block）

  • 用三个以上连续的`或~.字符作为标记（如```或~~~）
  • 开始的标记后，可以有可选的info string，可以指定高亮语言类型
  • 如果开始行有缩进，那么每行内容都会去掉与开始行相同数量的缩进。如果内容行缩进比较少，则去掉所有缩进
  • 开始行和结束行都可以有不多于三个空格的缩进，两者的缩进可以不同
  • 代码块中的内容不会被解释
  • 结束行需要包含不少于开始行的标记符号
```python
print("hello, world!")
```

注释

  • markdown中没有专门的注释语法，一般用HTML注释，但会出现在最终结果中。 
    <!-- html comment -->
  • 可以用引用代替注释，只要不使用，是不会出现在结果中的： 
    [//]: # (This may be the most platform independent comment)
